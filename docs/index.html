What this is

A working Apache Iceberg lakehouse that runs on a laptop. It loads the MovieLens dataset and a synthetic stream of playback events, moves them through bronze, silver, and gold layers with dbt, and serves the result through Trino. I built it to get hands-on with Iceberg, the open table format most modern data platforms are standardizing on, and to show what it buys you over a folder of Parquet files: reading a table as it looked last week, changing a schema without rewriting data, and partitioning you never have to manage by hand. Every number on this page came from one real run.

How the data flows

Bronzeraw ingest
→
Silverdims + facts
→
Goldmarts
←
TrinoSQL

PySpark writes Iceberg tables to MinIO (S3) through a REST catalog. dbt builds the raw bronze tables into clean silver dimensions and facts, then into the gold marts the charts read from. Trino queries those same tables directly. The gold layer only counts as good once the dbt tests and the Great Expectations checks pass.
